Some technologies I've worked with

Hover over an icon to read about it!
Flutter is currently my choice framework for mobile app development.
It's powerful, well-suported and lends itself well to the philosophy of letting the developer control the architecture. I find it to be quite unopinionated as far as frameworks go and I can prototype apps in it very quickly.
As of right now, NestJS is my favourite backend JS framework.
The documentation is made to get you going now with as little friction as possible. Parts of it are inspired by Angular and I often joke that NestJS explains Angular better than the actual Angular documentation! While I can't call myself an expert in it, I have used it on multiple smaller projects.
Most of my current job revolves around Dotnet.
I use ASP.NET for API development and Blazor for frontend development. I have spent months at this point grinding on systems that are built on Dotnet and C# and feel very comfortable with it.
We did heavy C++ programming during our studies at university.
Recently I haven't used it as much, since I'm mostly focused on web development. I still try to keep up with it from time to time, as I enjoy lower-level programming.
When I have a choice I much prefer using Linux over Windows, having found that it's a much more inviting and developer-friendly environment. I've gotten used to it over the last few years and I'm always looking to sharpen my 'Unix skills', since there's still a lot I don't know or understand. I also have experience using the Unix-based MacOS.
Most of my Python knowledge stems from my study of machine learning.
I've learned quite a bit of python while working on neural networks in PyTorch and I've grown relatively accustomed to the 'python way' though I seldom use Python for things like API development.
Rust is something that I've been meaning to sink some more time into.
While I have started to gain a feeling for the language, I am still very far from proficiency. Since I've decided to try and learn it, I've found that using it is a frustrating but very educational experience.
I have experience using Docker for containerisation. In particular I've used Docker Compose on several occasions. I have not used it extensively but I have started to understand its philosophy:
Having to manually keep track of several different parts of a system (api, db, messaging queue....) can become frustrating and error prone. Deploying everything 'under one roof' in a Docker Compose is much neater.
I do have experience using Java, mostly during my studies.
I find that it's very nice for making portable code (all you need is a JVM to run it after all) but it's never actually come up during my work in industry.
I have a ton of experience with similar languages like C# and Dart, so I don't feel too out of place with Java.
